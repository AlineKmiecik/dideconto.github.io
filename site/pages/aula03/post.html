O QUE É O NODEMON?

Nodemon é um utilitário que serve para auxiliar no desenvolvimento de uma aplicação. Ele monitora mudanças no código-fonte. Sua principal vantagem é automaticamente reiniciar o servidor.

COMO INSTALAR O NODEMON NO MEU PROJETO?

No terminal do VS, digite o comando “npm install nodemon –save-dev” (o “—save-dev” é para no package.json ele for instalado como dependência de desenvolvimento e não uma global, como o express). Ou digite “npm install -g nodemon” para instalar o instalar como global;

Fazendo a conexão com o MongoDB Cloud

Após ter feito o cadastro no MongoDB e Cluster ter sido criada clicamos no "Connect" para iniciarmos a conexão.

Irão aparecer 3 opções, a primeira serve para que apenas pessoas de uma região do mundo consigam mexer, trabalhar, editar, etc. Selecionamos a primeira opção “Add your Current IP Address” e ele vai puxar automaticamente o seu IP.

No segundo passo iremos criar um nome e uma senha para o banco de dados. Com o nome e a senha clicamos em “Create Database User”.

Com o IP selecionado e o banco de dados criado selecionamos a opção “Choose a connection method” selecionamos o “Connect your aplication”.

Selecionamos a versão e a aplicação que está sendo utilizada.

(Opcional) O MongoDB Compass é utilizado também para verificarmos o que está acontecendo na aplicação sem precisar usar o terminal. Para baixá-lo acessamos esse link https://www.mongodb.com/try/download/compass. Tanto faz qual você irá querer utilizar, ambos possuem a mesma função. Após ter baixado o MongoDB Compass e abri-lo colocamos o link da imagem acima.

Após colocarmos a URL devemos substituir onde está escrito <"password"> pela senha que foi criada quando estávamos criando o banco de dados, não a senha de cadastro utilizada para acessar o MongoDB. E também substituir <"dbname"> pelo nome criado na criação do banco. Após ter substituído clicamos em “Connect”. Com a conexão feita voltamos ao Visual Studio Code e colamos a URL de conexão na variável que está fazendo a conexão. Lembrando que devemos substituir o <"password"> e <"dbname">.
