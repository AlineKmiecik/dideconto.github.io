Introdução ao Express

--- Teórico ---

Express é o framework Node mais conhecido e uma das bibliotecas mais profundas para uma série de outros frameworks do Node. Algumas de suas soluções são:

  • Integrar "view engines" para inserir dados nos templates.
  • Definir as configurações comuns da aplicação web, como a porta a ser usada para conexão e a localização dos modelos que são usados para renderizar a resposta.

--- Prático ---

Com a compilação do código, ocorre que é definido o uso de recursos do express e com isso cria uma aplicação na porta 1234, que com ela é possível fazer requisições tendo uma resposta a mesma, sendo neste caso a frase "Primeira aplicação com NodeJs", além de retornar uma frase no console do próprio Visual Studio Code, sendo essa "Servidor rodando na porta 1234".


--- Git Ignore ---

O arquivo .gitignore se comunica com o git para dizer quais arquivos/diretórios ele deve ignorar ao fazer um commit. Para compartilhar as mesmas regras com outros usuários que clonam o repositório, basta enviar o .gitignore junto para seu repositório.
